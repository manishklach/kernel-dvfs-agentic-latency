Systems Research Project

kernel-dvfs-agentic-latency

Agentic AI control loops are bursty: they issue work, block in a gate like epoll_wait() or io_uring_enter(), wake on a completion, do a small amount of parse-and-decide CPU work, then block again. On ARM Linux, that shape can repeatedly pay cpuidle exit and DVFS ramp latency.

This repository packages an RFC kernel patch, tracing tools, a microbenchmark, diagrams, and a measurement plan around one narrow hypothesis: a short bounded agent-active window may reduce repeated warmup penalties without replacing Linux scheduler or power-management policy.

Research / RFC only. This is not production code, not upstream-ready, and not a claim of benchmark wins.

Patch Summary

The RFC introduces CONFIG_AGENT_LATENCY and SCHED_FLAG_AGENT_LATENCY, adds small task and runqueue metadata, refreshes a short active window on wakeup, raises effective util in schedutil, and guards against deep idle exits during that window.

What It Is Not

This does not replace CPUFreq, schedutil, uclamp, cpuidle, or fair scheduling. It does not define a new RT class, and it does not yet solve policy, cgroup budgets, thermal constraints, or abuse prevention.

Why It Matters

Agent loops can amplify small per-step delays across hundreds or thousands of iterations. The interesting question is whether wakeup-to-run latency, idle exit, and frequency ramp dominate user-visible time more than aggregate throughput does.

Measurement Plan

  • Baseline with schedutil and default cpuidle
  • performance governor as a latency ceiling
  • uclamp.min as an existing mechanism
  • Patched kernel with CONFIG_AGENT_LATENCY=y
  • Future work: IRQ-aware placement

Key Metrics

  • Step latency p50/p99/p999
  • Wakeup-to-run latency
  • CPU frequency transitions
  • Idle-state residency
  • epoll_wait() and io_uring_enter() blocked duration
  • Derived frequency-ramp-after-wakeup metric

Quickstart

git am patches/0000-agent-latency-dvfs-rfc.patch
scripts/config --enable AGENT_LATENCY
make -C selftests
./selftests/agent_loop --steps 10000 --interval-us 500
./selftests/agent_loop --agent-latency --steps 10000 --interval-us 500
sudo bpftrace tools/bpftrace/wakeup_to_run.bt
sudo bpftrace tools/bpftrace/cpu_frequency.bt
